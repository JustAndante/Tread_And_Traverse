# Track Physics Solver: справочник настроек

Этот справочник повторяет порядок релизных параметров компонента
`Track Physics Solver` в панели Details сверху вниз. Он описывает основной
рабочий процесс на основе костей ходовой, полезную диагностику и штатные
настройки физики, вывода и производительности.

Названия параметров сохранены на английском ровно так, как они отображаются в
Unreal Engine. Значения ниже — исходные значения класса. При выборе
`Physics Feel Preset` компонент сразу применяет набор настроек выбранного
пресета, поэтому на уже настроенной машине цифры могут отличаться. Зависимые
поля скрываются, когда выключена родительская функция. Если не указано иное,
расстояния заданы в сантиметрах Unreal.

Справочник сосредоточен на поддерживаемом релизном процессе настройки машины
по костям ходовой.

## Output: целевой сплайн

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Target Spline` | None | Имя компонента сплайна, в который записывается рассчитанный замкнутый контур гусеницы. Укажите тот же сплайн, который использует соответствующий `Track Spline Builder`. |
| `Target Spline Override` (Advanced) | None | Прямая ссылка на компонент, имеющая приоритет над поиском по имени. Обычно поле оставляют пустым и используют `Target Spline`; оно нужно для нестандартного владения компонентами или программной настройки. |

## Setup: источник геометрии

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Closed Loop` (Advanced) | On | Соединяет первую и последнюю рассчитанные точки. Непрерывная танковая гусеница должна оставаться замкнутой. |
| `Generate From Bone Rig` | On | Строит полный контур управляющих точек по костям опорных катков, поддерживающих роликов и концевых колёс. Для релизной настройки машины оставляйте параметр включённым. |
| `Source Mesh Component` | None | Skeletal Mesh или Scene Component, содержащий настроенные кости колёс. Он же служит достоверным источником движения подвески во время симуляции. |
| `Side Preset` | Auto | Заполняет имена для левой или правой стороны. `Left` и `Right` используют соответствующие имена костей, `Auto` пытается определить сторону, а `Custom` сохраняет ручные имена. Ручные правки имеют приоритет. |
| `Auto Detect Wheel Counts` | On | Ищет последовательно пронумерованные кости и останавливается после окончания ряда опорных катков или верхних роликов. |
| `Wheel Auto-Detection Limit` (Advanced) | 32 | Максимальное число индексов, проверяемых для каждого префикса опорных катков и верхних роликов. Видно только при автоматическом определении количества. |

### Сгенерированные и сохранённые данные (только чтение)

Эти поля находятся рядом с настройками источника, но принадлежат самому
решателю и не являются обычными пользовательскими параметрами.

| Поле | Что означает |
| --- | --- |
| `Control Points` | Описания управляющих точек, по которым строится желаемый контур. В обычном процессе массивом управляет генератор по костям. |

## Setup: опорные катки

| --- | ---: | --- |

| `Road Wheel Suffix` | Пусто | Текст после числового индекса. Оставьте пустым для имён вида `LM_Wheel_01`; суффикс нужен только тогда, когда он присутствует в риге. |
| `First Road Wheel Index` | 1 | Индекс первой кости опорного катка в последовательности. |
| `Road Wheel Count` | 7 | Количество опорных катков при ручной настройке. Видно только при выключенном `Auto Detect Wheel Counts`. |
| `Road Wheel Index Digits` (Advanced) | 2 | Минимальное количество цифр в индексе. Значение 2 создаёт имена с `01`, `02` и далее. |
| `Trace Ground Between Road Wheels` | On | Выполняет трассировку поверхности для нижних промежуточных точек между центрами соседних опорных катков. Выключайте только для жёсткой или чисто интерполированной нижней ветви. |
| `Road Wheel Radius` | 39 cm | Основной контактный радиус: по нему гусеница размещается под центрами катков и удерживается снаружи колёс физическими ограничениями. |

| Параметр | По умолчанию | Назначение |

| `Top Roller Bone Prefix` (Advanced) | `TopWheel` | Текст перед числовым индексом кости верхнего ролика. |
| `Top Roller Bone Suffix` (Advanced) | `_L` | Текст после индекса. Пресет стороны обновляет суффикс для левой или правой ходовой. |
| `First Top Roller Index` | 1 | Индекс первой кости верхнего ролика. |
| `Top Roller Count` | 4 | Количество верхних роликов при ручной настройке. Видно только при выключенном автоматическом определении количества. |
| `Top Roller Index Digits` (Advanced) | 2 | Минимальное количество цифр при составлении имён костей верхних роликов. |
| `Top Roller Contact Radius` | 22 cm | Радиус, по которому гусеница огибает верхний ролик и соблюдает одностороннее ограничение его окружности. |
| `Allow Top Roller Detach` | Off | Разрешает верхним точкам отходить от точного положения кости, сохраняя ограничение радиусом. Верхняя ветвь получает возможность реагировать на удары и инерцию подвески. |
| `Top Run Physics` | On | Включает физическую симуляцию верхней возвратной ветви. Выключение оставляет более дешёвую процедурную форму. |
| `Upper Run Physics Weight` (Advanced) | 0.45 | Сила физического движения и инерции для отсоединённых верхних точек. Видно только при одновременно включённых верхней физике и отсоединении. |
| `Detached Upper Travel Limit` (Advanced) | 18 cm | Максимальное отклонение отсоединённых верхних точек от созданной формы. Значение 0 отключает это ограничение. |
| `Speed Tension` | Off | Уменьшает провисание верхней ветви с ростом скорости машины, не меняя нижнюю контактную форму. Точная реакция задаётся выбранным физическим пресетом. |

## Setup: концевые колёса

Раздел `End Wheels` охватывает переднее направляющее или концевое колесо и
заднее ведущее колесо. Это условные механические роли: на другой машине ведущим
может быть любой конец, если кости, радиусы и порядок контура настроены верно.

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Generate End Rollers` | On | Строит переднюю и заднюю дуги концевых колёс, соединяющие верхнюю и нижнюю ветви. |
| `Front End Wheel Bone` | `LF_Wheel_00` | Кость, используемая как центр передней дуги обвода. |
| `Rear Drive Wheel Bone` | `LB_Wheel_08` | Кость центра задней дуги и, по умолчанию, источник дистанции прокрутки визуальной гусеницы. |
| `Front End Wheel Wrap Radius` | 55 cm | Радиус дуги центра трака вокруг переднего концевого колеса. |
| `Rear Drive Wheel Wrap Radius` | 0 cm | Радиус заднего ведущего колеса. Значение 0 использует `Front End Wheel Wrap Radius`. |
| `Wrap Points Per End Wheel` | 3 | Количество основных управляющих точек на каждой концевой дуге. Большее значение точнее описывает окружность, но увеличивает объём расчёта. |
| `End Wheel Wrap Arc` (Advanced) | 130 degrees | Угловая длина части окружности, используемой для построения обвода каждого концевого колеса. |
| `End Wheel Wrap Angle Offset` (Advanced) | 0 degrees | Поворачивает обе дуги вокруг центров колёс, позволяя совместить переходы с конкретным ригом. |

## Setup: детализация формы

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Generate Detail Points By Spacing` | On | Добавляет промежуточные точки на длинных участках, чтобы плотность зависела от заданного расстояния, а не только от количества основных точек колёс. |
| `Target Detail Point Spacing` | 35 cm | Желаемый максимальный шаг промежуточных точек. Меньшее значение создаёт более плотный и более дорогой сплайн. |
| `Maximum Detail Points Per Segment` (Advanced) | 6 | Защитный предел количества автоматически создаваемых точек между двумя соседними основными контролами. |
| `Trace Generated Detail Points` | On | Разрешает промежуточным точкам искать поверхность, а не только интерполироваться между основными контролами. Важно для ступеней и препятствий между катками. |
| `Unsupported Span Sag Ratio` | 0.08 | Глубина провиса как доля длины неподдерживаемого участка. Также масштабирует слабину физической цепи и предел дополнительного провиса, поэтому крупные изменения влияют и на желаемую форму, и на физическую реакцию. |

## Setup: трассировка поверхности

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Enable Ground Tracing` | On | Разрешает созданным точкам с включённым trace находить поверхности мира и формировать контакты нижней ветви. |
| `Snap Trace Points To Ground` | On | Помещает принятую точку прямо на результат контакта. При выключении используется `Ground Alignment Strength` для более мягкого сближения. |
| `Ground Trace Length` | 350 cm | Полная длина поиска вниз для автоматически созданных trace-точек. |
| `Ground Trace Start Height` | 120 cm | Высота начала трассировки над созданной точкой. |
| `Track Contact Width` | 0 cm | Дополнительная поперечная ширина контактной области башмака. Значение 0 использует только центральный probe; значение около ширины башмака включает трёхточечный контактный manifold. |
| `Ground Trace Shape` (Advanced) | Sphere | `Line` использует луч нулевого радиуса. `Sphere` выполняет sweep заданного радиуса и лучше переносит мелкие щели и резкие детали меша. |
| `Ground Trace Radius` (Advanced) | 3 cm | Радиус сферы при выбранной форме `Sphere`. |
| `Ground Alignment Strength` (Advanced) | 0.35 | Сила сближения с принятой поверхностью, когда прямое прилипание `Snap Trace Points To Ground` выключено. |
| `Maximum Ground Snap Distance` (Advanced) | 26 cm | Отклоняет контакт, если для его принятия точку нужно переместить дальше этого расстояния. Защищает от захвата далёкой или посторонней поверхности. |
| `Maximum Ground Search Drop` (Advanced) | 90 cm | Отклоняет поверхность, найденную слишком низко относительно созданной точки, даже если она находится внутри полной длины trace. |

`Ground Trace Length` определяет, где решатель может искать поверхность.
`Maximum Ground Snap Distance` и `Maximum Ground Search Drop` определяют,
какой найденный результат разрешено принять. Эти параметры решают разные
задачи и не заменяют друг друга.

## Статус решателя (только чтение)

Поля статуса показывают результат последней генерации или итерации решателя и
не меняют гусеницу. Начинайте диагностику с трёх текстовых полей. Раскрывайте
расширенные категории только для поиска конкретной проблемы.

### Основной статус

| Поле | Что означает |
| --- | --- |
| `Last Solver Status` | Читаемая сводка последней генерации, расчёта, контактов, физического режима, обновления Builder и ключевых диагностических значений. |
| `Last Validation Status` | Результат последней проверки настройки: отсутствующий сплайн, источник, Builder или ошибки геометрии. |
| `Last Debug Snapshot` | Последний подробный снимок диагностики. Точная атрибуция пиков требует включённого `Detailed Runtime Diagnostics`. |
| `Last Desired Point Count` | Количество точек, созданных стадией желаемой формы. |
| `Last Solved Point Count` | Количество точек в окончательном расчёте. Обычно должно совпадать с желаемым количеством. |
| `Last Missing Source Count` | Количество контролов, для которых не удалось найти компонент, кость или socket. В норме — 0. |
| `Last Ground Contact Count` | Количество желаемых точек, которые сейчас приняты как контакты с поверхностью. |
| `Last Matched Wheel Points` | Количество созданных точек, успешно сопоставленных с костями и guide-окружностями колёс. |
| `Last Generated Control Point Count` | Количество основных контролов после последней генерации. |
| `Last Skipped Endpoint Wheels` | Концевые кости, намеренно исключённые из ряда опорных катков, поскольку ими управляет генератор концевых колёс. |
| `Last Desired Extent` | Примерный пространственный размер созданного контура, используемый проверкой и диагностикой. |
| `Last Bad Segment Count` | Количество сегментов, отклонённых защитой от плохой геометрии. В норме — 0. |
| `Last Max Segment Length` | Самый длинный сегмент последнего рассчитанного контура. Неожиданный пик указывает на сломанную топологию или скачок точки. |

## Действия в редакторе

Действия отображаются как моментальные галки. После выполнения компонент сам
сбрасывает галку.

| Действие | Назначение |
| --- | --- |
| `Generate Bone Rig Now` | Повторно создаёт основной контур из текущего источника, стороны, имён костей, радиусов, обвода, формы и trace-настроек. Используйте после изменения топологии или имён костей. |
| `Preview Solved Spline After Generate` (только чтение) | Показывает, настроена ли генерация в редакторе на сохранение решённого preview в целевом сплайне. Обычная генерация сохраняет стабильный авторский или созданный контур. |

## Simulation: физическое поведение

Изменение `Physics Feel Preset` применяет полный набор физических значений и
сбрасывает историю решателя. Сначала выберите ближайший по поведению пресет, а
затем при необходимости вносите небольшие изменения в открытые параметры.

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Physics Feel Preset` | Realistic Heavy | Выбирает общий режим решателя и согласованный набор физических параметров. |
| `Responsive` | Вариант пресета | Быстро следует цели, даёт плотную стабильную реакцию и минимум свободной инерции. Самый дешёвый и наименее свободный физически пресет. |
| `Realistic Heavy` | Вариант пресета | Использует физическую цепь с умеренной слабиной, сильными guide-ограничениями колёс, ground-контактами, реакцией подвески и тяжёлым демпфированием. Рекомендуемая отправная точка для танка. |
| `Loose Physical` | Вариант пресета | Даёт больше слабины, инерции, движения отсоединённой верхней ветви и слабее следует форме. Выразительнее, но проще теряет устойчивость на экстремальной геометрии. |
| `Chain Gravity Scale` | 1.0 | Множитель гравитации во время интеграции физической цепи. Выбранный пресет обычно заменяет исходное значение. |
| `Generated Shape Pull` | 0.35 | Притягивает симулируемые точки к желаемой форме. Большее значение лучше контролирует форму; меньшее отдаёт приоритет контактам, constraints и инерции. |
| `Wheel Guide Pull` (Advanced) | 0.10 | Минимальное притяжение к формам верхних и концевых колёс. Отсоединённые верхние точки получают уменьшенную долю. |
| `Per-Segment Slack Ratio` | 0 | Добавляет податливую слабину rest length каждому физическому сегменту. Итоговая слабина также масштабируется `Unsupported Span Sag Ratio`. |
| `Chain Velocity Damping` (Advanced) | 0.15 | Гасит скорость физических точек на каждом шаге. Большие значения быстрее успокаивают движение; чрезмерные делают гусеницу жёсткой. |
| `Preserve Ground Contacts` (Advanced) | On | Сохраняет принятые ground-контакты во время решения constraints вместо немедленного отрыва цепи. |
| `Ground Contact Strength` (Advanced) | 1.0 | Сила коррекции сохранённого контакта. Видно только при включённом сохранении контактов. |
| `Maximum Extra Sag` | 35 cm | Максимальное дополнительное движение вниз относительно процедурной формы неподдерживаемого участка. Значение 0 отключает предел. |
| `Keep Track Outside Wheels` | On | Применяет односторонние радиусные ограничения опорных, верхних и концевых колёс. Не даёт физическим точкам пройти сквозь колесо, но не притягивает уже свободные точки обратно к окружности. |
| `Wheel Surface Following` (Advanced) | 0 | Направляет близкие физические точки вдоль окружности после радиусного ограничения. Рекомендуемый пресет задаёт собственное ненулевое значение. |

## Runtime

| Параметр | По умолчанию | Назначение |
| --- | ---: | --- |
| `Rebuild On Begin Play` | On | Планирует первоначальную runtime-генерацию и перестроение после BeginPlay. |
| `Runtime Quality` | Full Physical | `Full Physical` выполняет обычные traces и физический расчёт. `Cheap Remote` отключает дорогую физику и trace-детализацию, оставляя ключевую процедурную форму для удалённых или реплицируемых визуальных копий. |
| `Lower Shoe Tangent Scale` | 0.55 | Длина tangent нижнего участка башмаков в режиме Cheap Remote. Видно только в `Cheap Remote`. |

| `Max Solve Delta Time` (Advanced) | 0.05 s | Максимальный временной шаг одного расчёта. Более длинные накопленные кадры ограничиваются, защищая физику от резкого скачка. |

## Как использовать этот справочник

Этот релизный справочник следует дополнить коротким практическим Quick Start:

1. подключение компонентов и настройка левой/правой стороны;
2. генерация и проверка результата;
3. радиусы опорных, верхних и концевых колёс, а также ширина гусеницы;
4. выбор физического пресета;
5. количество звеньев, ориентация и привод в Track Builder;
6. чтение debug и status-полей;
7. производительность и режимы качества для multiplayer.
